# LogCollectorApp — сборщик логов СЭД

## Назначение

LogCollectorApp — настольное приложение для автоматизированного сбора логов с серверов системы электронного документооборота.

Программа помогает быстро получить фрагменты логов за нужный период без ручного подключения к каждому серверу. Пользователь выбирает группу серверов, отмечает нужные серверы, задает начало и конец поиска, после чего приложение подключается к выбранным узлам, скачивает подходящие лог-файлы, ищет записи за указанный интервал и сохраняет результат в выбранную папку.

Приложение предназначено для работы в закрытой сети, где доступны сервер PostgreSQL и целевые серверы Astra Linux по SSH/SFTP.

## Что умеет программа

- загружать из базы данных группы серверов и серверы внутри выбранной группы;
- добавлять и редактировать группы серверов;
- добавлять и редактировать серверы внутри группы;
- сохранять SSH-логин и SSH-пароль локально на компьютере пользователя;
- выбирать папку для сохранения результатов;
- собирать логи за указанный период с одного или нескольких серверов;
- работать с текущими логами и архивными логами для поддерживаемых групп серверов;
- показывать текущий статус выполнения операции;
- продолжать обработку остальных серверов, если на одном сервере произошла ошибка;
- сохранять найденные записи в отдельные `.log`-файлы.

## Требования для работы

Для запуска приложения пользователю нужен компьютер с Windows x64.

Отдельно устанавливать .NET Runtime не требуется: установщик уже содержит необходимые компоненты для запуска приложения.

Для полноценной работы должны быть доступны:

- сервер PostgreSQL с заполненными справочниками групп и серверов;
- сеть, в которой доступна база данных и серверы с логами;
- SSH/SFTP-доступ к серверам, с которых выполняется сбор логов;
- папка на компьютере пользователя, куда можно сохранять результаты.

Если приложение запускается вне закрытой сети или без доступа к базе данных, список групп и серверов не загрузится.

## Установка

Готовый установщик находится в папке проекта:

```text
LogCollectorApp\dist\LogCollectorApp_Setup_1.0.0.exe
```

Для установки выполните следующие действия:

1. Откройте папку `LogCollectorApp\dist`.
2. Запустите файл `LogCollectorApp_Setup_1.0.0.exe`.
3. Следуйте шагам мастера установки.
4. При необходимости включите создание ярлыка на рабочем столе.
5. После завершения установки запустите приложение из меню «Пуск» или через созданный ярлык.

По умолчанию приложение устанавливается в профиль текущего пользователя. Права администратора обычно не требуются.

## Первый запуск

После первого запуска необходимо выполнить первичную настройку.

### 1. Настройка SSH-доступа

1. Нажмите кнопку **«Настройки»**.
2. Введите SSH-логин.
3. Введите SSH-пароль.
4. Нажмите **«Сохранить»**.

Эти данные используются для подключения к серверам при сборе логов. Логин и пароль сохраняются локально на компьютере пользователя.

### 2. Выбор папки для результатов

1. Нажмите кнопку **«Папка вывода»**.
2. Нажмите **«Обзор...»**.
3. Выберите папку, в которую нужно сохранять найденные логи.
4. Нажмите **«Сохранить»**.

Если папка вывода не выбрана, приложение использует папку `Output` рядом с рабочей директорией программы.

## Работа с группами серверов

Группа серверов объединяет несколько серверов одного типа или назначения. Например, это могут быть web-серверы или серверы приложений.

### Добавление группы

1. Нажмите **«+ Группа»**.
2. Укажите название группы.
3. При необходимости заполните описание.
4. Оставьте флажок **«Активна»**, если группа должна отображаться и использоваться.
5. Нажмите **«Сохранить»**.

### Редактирование группы

1. Выберите группу в списке **«Группа серверов»**.
2. Нажмите **«✎ Группа»**.
3. Измените название, описание или признак активности.
4. Нажмите **«Сохранить»**.

Если группа отключена, она считается неактивной и не должна использоваться для обычного сбора логов.

## Работа с серверами

После выбора группы в таблице отображаются серверы, относящиеся к этой группе.

В таблице доступны основные поля:

| Поле | Назначение |
| --- | --- |
| Выбрать | Отметка сервера для сбора логов |
| Имя сервера | Понятное название сервера |
| IP-адрес | Адрес сервера для подключения |
| Порт SSH | Порт SSH/SFTP, обычно `22` |
| Активен | Признак использования сервера |

### Добавление сервера

1. Выберите группу серверов.
2. Нажмите **«+ Сервер»**.
3. Укажите название сервера.
4. Укажите IP-адрес.
5. Укажите SSH-порт. Обычно используется порт `22`.
6. Оставьте флажок **«Активен»**, если сервер должен использоваться.
7. Нажмите **«Сохранить»**.

### Редактирование сервера

Изменить сервер можно двумя способами.

Первый способ — через отдельное окно:

1. Выберите сервер в таблице.
2. Нажмите **«✎ Сервер»**.
3. Измените нужные данные.
4. Нажмите **«Сохранить»**.

Второй способ — прямо в таблице:

1. Измените имя сервера, IP-адрес, порт или признак активности.
2. После завершения редактирования программа сохранит изменение в базе данных.

Если введен некорректный IP-адрес или порт, приложение покажет сообщение об ошибке и не сохранит изменения.

## Сбор логов

Перед запуском сбора убедитесь, что:

- выбрана группа серверов;
- в таблице отмечен хотя бы один сервер;
- SSH-логин и пароль сохранены в настройках;
- указана корректная папка вывода;
- рабочая станция имеет доступ к базе данных и выбранным серверам.

Порядок запуска:

1. В поле **«Группа серверов»** выберите нужную группу.
2. В таблице отметьте серверы, с которых нужно собрать логи.
3. В поле **«Начало поиска»** укажите дату и время начала периода.
4. В поле **«Конец поиска»** укажите дату и время окончания периода.
5. Нажмите **«Начать сбор логов»**.
6. Дождитесь завершения обработки.

Формат даты и времени:

```text
ДД.ММ.ГГГГ ЧЧ:ММ
```

Пример:

```text
19.06.2026 15:00
```

Конечное время должно быть больше начального.

## Результат работы

После завершения сбора приложение показывает итоговое сообщение:

- сколько серверов обработано успешно;
- по скольким серверам возникли ошибки;
- какие файлы были созданы;
- в какой папке сохранены результаты.

После закрытия итогового сообщения программа открывает папку с результатами в Проводнике Windows.

Результаты сохраняются в виде `.log`-файлов. Имя файла содержит название сервера и период поиска. Например:

```text
server01_20260619_150000_20260619_160000.log
```

Если по серверу не найдены записи за выбранный период, файл для этого сервера не создается, а информация отображается в итоговом сообщении.

Если один сервер недоступен или на нем возникла ошибка, сбор по остальным выбранным серверам продолжается.

## Где хранятся настройки

Пользовательские настройки сохраняются локально в профиле Windows:

```text
%APPDATA%\LogCollectorApp\settings.json
```

В этом файле хранятся:

- папка вывода;
- SSH-логин;
- SSH-пароль.

Обычно открывать или редактировать этот файл вручную не требуется. При необходимости настройки можно изменить через кнопки **«Настройки»** и **«Папка вывода»** внутри приложения.

## Удаление приложения

Удалить приложение можно стандартным способом Windows:

1. Откройте **Параметры** Windows.
2. Перейдите в раздел **Приложения**.
3. Найдите `LogCollectorApp`.
4. Нажмите **«Удалить»**.

Локальный файл настроек в `%APPDATA%\LogCollectorApp` может остаться после удаления. Если требуется полностью очистить настройки пользователя, удалите эту папку вручную.

## Частые проблемы

### Не загружаются группы серверов

Возможные причины:

- нет доступа к закрытой сети;
- сервер PostgreSQL недоступен;
- база данных не настроена;
- учетные данные подключения к базе данных неверны;
- в базе данных нет активных групп серверов.

Что сделать:

1. Проверьте подключение к рабочей сети.
2. Убедитесь, что сервер PostgreSQL доступен.
3. Обратитесь к администратору базы данных или ответственному за настройку справочников.

### Кнопка «Начать сбор логов» недоступна

Обычно это происходит, если не выбрана группа серверов или в выбранной группе нет серверов.

Что сделать:

1. Выберите другую группу.
2. Добавьте сервер в текущую группу.
3. Проверьте, что сервер активен.

### Программа просит настроить SSH-логин и пароль

Это означает, что локальные SSH-настройки еще не сохранены.

Что сделать:

1. Нажмите **«Настройки»**.
2. Введите SSH-логин и пароль.
3. Нажмите **«Сохранить»**.
4. Повторите запуск сбора.

### Ошибка подключения к серверу

Возможные причины:

- сервер выключен или недоступен по сети;
- указан неверный IP-адрес;
- указан неверный SSH-порт;
- введены неверные SSH-логин или пароль;
- на сервере отсутствует нужный лог-файл или архив.

Если ошибка возникла только на одном сервере, приложение продолжит обработку остальных выбранных серверов.

### Записи за период не найдены

Это не всегда ошибка. Такое сообщение означает, что приложение подключилось к серверу, но не нашло строк логов, соответствующих выбранному периоду.

Что сделать:

1. Проверьте правильность начального и конечного времени.
2. Убедитесь, что выбран правильный день.
3. Попробуйте расширить период поиска.
4. Проверьте, что выбрана правильная группа серверов.

### Результаты не появились в выбранной папке

Возможные причины:

- по выбранному периоду не найдено записей;
- нет прав на запись в выбранную папку;
- папка находится на недоступном сетевом диске;
- операция завершилась ошибкой до создания файлов.

Что сделать:

1. Выберите локальную папку, например `Документы\LogCollector`.
2. Повторите сбор.
3. Проверьте итоговое сообщение после завершения операции.

## Рекомендации по использованию

- Перед большим сбором проверьте работу на одном сервере.
- Не выбирайте лишние серверы, если нужны логи только с одного узла.
- Указывайте максимально точный период поиска: это сокращает время обработки.
- Не закрывайте приложение во время сбора логов.
- После изменения SSH-пароля обязательно обновите его в настройках приложения.
- При ошибках подключения сначала проверьте сеть, затем IP-адрес, порт и SSH-данные.
